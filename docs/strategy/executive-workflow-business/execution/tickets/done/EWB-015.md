## Description
Research, compare, and recommend the best headed-browser MCP tools for full UI automation by LLM agents. The goal is to eliminate all "needs manual browser work" blockers across the ticket backlog.

## Tools Assessed

### Currently Available (Hands-On Tested)
1. **Kapture MCP** — Chrome/Edge DevTools panel approach. Click, fill, navigate, screenshot, DOM query. Tested on Porkbun DNS, Firebase Console, Tally.
2. **Playwright MCP** (`@playwright/mcp`) — Microsoft's official MCP server. Headed browser, accessibility tree snapshots, click/type/navigate. Tested on Cal.com, Tally, Google sign-in.
3. **Puppeteer (Cline built-in)** — `browser_action` tool. Launch, click coordinates, type, screenshot, scroll. Coordinate-based.

### Researched (Docs/Repos Only)

6. **Stagehand** (Browserbase) — Natural language actions
7. **AgentQL** — Semantic web scraping
8. **Steel MCP** — Headless browser sessions
9. **Chrome DevTools MCP** — Direct CDP integration
10. **Operative** — Visual grounding framework

## Recommendation

**Primary: Playwright MCP** — Best for structured form filling, accessibility tree snapshots, and multi-step flows. Works well with Claude/Goose tool interface.

**Fallback: Kapture MCP** — Best for tasks requiring existing browser sessions (authenticated portals like Firebase Console, Porkbun). Uses user's logged-in state directly.

**Finding from EWB-013 execution:** For many vendor integrations, REST APIs (Cal.com v2, Tally API) proved faster and more reliable than browser automation. The recommendation hierarchy is: **CLI > API > Playwright > Kapture > Manual**.
