## Description
Research, compare, and recommend the best headed-browser MCP tools for full UI automation by LLM agents. The goal is to eliminate all "needs manual browser work" blockers across the ticket backlog. Every remaining ticket that requires UI interaction (HubSpot pipeline setup, LinkedIn profile editing, Cal.com configuration, Firebase console setup, Tally form creation) should be automatable through the selected tool(s).

## Tools To Assess

### Currently Available
1. **Kapture MCP** — Chrome/Edge DevTools panel approach. Already connected. Click, fill, navigate, screenshot, DOM query.
2. **Playwright MCP** (`@playwright/mcp` / `github.com/microsoft/playwright-mcp`) — Microsoft's official MCP server. Headed browser, accessibility tree snapshots, click/type/navigate.
3. **Puppeteer (Cline built-in)** — `browser_action` tool. Launch, click coordinates, type, screenshot, scroll. Coordinate-based.

### To Research
4. **Computer Use MCP** (Anthropic) — Screen-level pixel control. Click, type, screenshot at OS level. Works with any app.
5. **Browser Use** (`browser-use/browser-use`) — Open source browser automation specifically for AI agents. Semantic element selection.
6. **Stagehand** (Browserbase) — AI-first browser automation. Natural language actions. `act()`, `extract()`, `observe()` API.
7. **AgentQL** — Semantic web scraping and interaction using natural language queries.
8. **Steel MCP** — Headless browser MCP server for AI agents with session management.
9. **Chrome DevTools MCP** — Direct Chrome DevTools Protocol integration.
10. **Operative** — AI browser automation framework with visual grounding.

## Evaluation Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| LLM Agent Compatibility | 0.25 | Works well with Cline/Claude, structured tool interface, good error handling |
| UI Interaction Reliability | 0.20 | Can reliably fill forms, click buttons, navigate multi-step flows, handle dynamic content |
| Authentication Handling | 0.15 | Can work with OAuth flows, session cookies, logged-in state persistence |
| Headed Browser Support | 0.10 | User can see what's happening in real-time (trust + debugging) |
| Selector Strategy | 0.10 | CSS vs. accessibility tree vs. semantic vs. visual — which is most reliable for LLMs? |
| Speed / Latency | 0.05 | Time per action, startup time, screenshot overhead |
| Open Source / Cost | 0.05 | Free tier availability, self-hostable, no vendor lock-in |
| MCP Integration Maturity | 0.10 | Official MCP server, well-maintained, active community |
