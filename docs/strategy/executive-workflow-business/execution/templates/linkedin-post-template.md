# LinkedIn Post Templates

## How To Use

1. Pick the format that matches your goal for the day
2. Fill in the brackets with your specific content
3. Run through the quality checklist at the bottom
4. Schedule via Buffer or post natively

---

## Format 1: Operator Observation (Monday)

Best for: Naming a specific workflow failure the ICP recognizes

```
[Hook: One specific workflow failure in 1–2 lines]

[3–5 lines expanding with concrete details — tools, steps, time wasted]

[1–2 lines on why this matters to a CIO/COO/Head of Ops]

[Optional: One-line reframe or insight]

[Question to drive comments from ICP]
```

### Example:

```
Most weekly leadership updates are assembled by hand.

Someone opens 4 tabs. Copies from Jira. Pastes into a doc. 
Checks Slack for context. Emails the draft. Gets feedback. Revises.

This takes 3–5 hours per cycle. Every week.

The tools are fine. The workflow between them is the problem.

What recurring report eats the most time on your team?
```

## Format 2: Framework / Teardown (Wednesday)

Best for: Showing diagnostic depth and systems thinking

```
[Hook: Name the framework or the thing being broken down]

[Numbered list or step-by-step breakdown]

[Key insight — where does it break?]

[Connect to the workflow wedge]

[CTA: 💾 Save this / Share if this resonates]
```

### Example:

```
The 7 hidden steps inside a weekly exec readout:

1. Gather tickets from Jira
2. Check Confluence for context docs
3. Scan Slack/Teams for updates that never made it to a system
4. Synthesize into a narrative
5. Add risk flags from memory (not data)
6. Send for review
7. Revise based on feedback that arrives too late

Steps 3–5 are where trust breaks down.

If the output is not source-linked, nobody believes it.

💾 Save this if your team has this problem.
```

## Format 3: Proof Post (Friday)

Best for: Demonstrating real operating credibility

```
[Hook: One-line outcome or credibility signal]

[2–3 lines on the problem context — what was broken]

[What was built or delivered — be specific]

[Measurable outcome or operating proof]

[Connect to the workflow offer without hard-selling]
```

### Example:

```
Reduced executive reporting prep from 6 hours to 45 minutes for a 
cross-functional ops team.

The problem: leadership readouts required pulling data from Jira, 
Confluence, and Teams — manually — every Monday morning.

The fix: a source-linked workflow that pulls, synthesizes, and 
formats the readout with human review before delivery.

The outcome: same quality, 80% less prep time, and the team 
actually trusts the output because every data point is linked 
back to its source.

This is what "workflow-first AI" looks like in practice.
```

## Pre-Publish Checklist

Before every post:

- [ ] Does this speak to a real ICP pain (not generic AI commentary)?
- [ ] Does it reinforce the workflow wedge?
- [ ] Does it signal real operator experience?
- [ ] Would a CIO/COO/Head of Ops find this valuable?
- [ ] Is the hook specific enough to stop scrolling?
- [ ] Is the language buyer-level (not technical jargon)?
- [ ] Does it avoid: AI hype, motivational fluff, tool lists, lifestyle content?
