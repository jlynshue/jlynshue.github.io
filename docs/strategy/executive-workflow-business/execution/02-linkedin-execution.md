### Monthly Cadence

- 12 standard posts (3/week × 4 weeks)
- 1 longer-form LinkedIn article or essay
- 1 offer-adjacent post explicitly tied to diagnostic or sprint

### Post Format Templates

#### Format 1: Operator Observation (Monday)

```
[Hook: Name a specific workflow failure in 1–2 lines]

[3–5 lines expanding the observation with specifics]

[1–2 lines on why this matters to the buyer]

[Optional: One-line framework or reframe]

[Soft CTA or question to drive comments]
```

**Example:**
```
Most weekly leadership updates are assembled by hand.

Someone opens 4 tabs. Copies from Jira. Pastes into a doc. 
Checks Slack for context. Emails the draft. Gets feedback. Revises.

This takes 3–5 hours per cycle. Every week.

The tools are fine. The workflow between them is the problem.

What recurring report eats the most time on your team?
```

#### Format 2: Framework / Teardown (Wednesday)

```
[Hook: Name the framework or the thing being torn down]

[Step-by-step or numbered breakdown]

[Key insight or counterintuitive point]

[Connect to the workflow wedge]

[CTA: save this / share if this resonates]
```

**Example:**
```
The 7 hidden steps inside a weekly exec readout:

1. Gather tickets from Jira
2. Check Confluence for context docs
3. Scan Slack/Teams for updates that never made it to a system
4. Synthesize into a narrative
5. Add risk flags from memory (not data)
6. Send for review
7. Revise based on feedback that arrives too late

Steps 3–5 are where trust breaks down.

If the output is not source-linked, nobody believes it.

💾 Save this if your team has this problem.
```

#### Format 3: Proof Post (Friday)

[Connect to the offer without hard-selling]

#### Format 4: Offer-Adjacent Post (1x/month)

## Phase 3: Engagement Protocol (Ongoing)

### Daily Engagement (15 minutes/day)

**Morning (10 minutes):**
1. Check notifications and reply to comments on your posts
2. Comment on 3–5 posts from ICP-title accounts (CIO, COO, Head of Ops, PMO)
3. Accept relevant connection requests

**Afternoon (5 minutes):**
1. Check DMs and respond to any warm conversations
2. If someone engaged meaningfully with your post, consider a DM

### Comment Strategy

When commenting on others' posts:
- Add a specific insight, not generic praise
- Reference your own operating experience where relevant
- Never pitch in comments — build familiarity
- Target posts from: CIOs, COOs, PMO leads, RevOps, BizOps operators

### DM Strategy

Move to DM only when:
- Someone comments with a specific pain that matches the ICP
- A target buyer visits your profile after engaging with content
- A warm contact engages with workflow-related content

**DM template:**
```
Hey [name] — saw your comment about [specific pain]. That resonates.

I've been working on exactly that kind of workflow challenge with ops teams. 

Happy to share what I've been seeing if you'd ever want to compare notes. 
No pitch — just curious if the pattern matches.
```

### Audience-Building Loops

From strategy:

1. **Insight → Comment → DM:** Publish sharp insight → attract operator comments → move relevant conversations to DMs
2. **Proof → Profile Visit → Site Visit:** Publish outcome proof → drive profile and website interest → convert through CTA
3. **Framework → Save → Future Conversion:** Publish useful frameworks → build familiarity → convert later

## Content Quality Filters

Before publishing any post, check:

- [ ] Does this speak to a specific workflow pain the ICP has?
- [ ] Does it reinforce the workflow wedge (not generic AI commentary)?
- [ ] Does it signal real operator experience (not theory)?
- [ ] Does it support the discovery call motion?
- [ ] Would the target buyer (CIO/COO/Head of Ops) find this valuable?

### Post More Of

- Operating pain with specific details
- Workflow diagrams described in words
- Concrete buyer language
- Trust and validation patterns
- Lessons from real systems work

### Post Less Of

- Generic AI optimism
- Vague motivational advice
- Broad futurism
- Tool screenshots without context
- Personal updates unrelated to the workflow wedge

### Never Post

- "10 AI tools you need today" style content
- Broad technical tutorials not tied to buyer pain
- Edge AI content that confuses the workflow offer
- Agency-style "we can build anything" messaging
- Lifestyle personal-brand content that weakens authority
