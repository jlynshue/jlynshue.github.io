## GitHub Execution

### Purpose

GitHub is not a demand generation engine. It is a supporting artifact for buyers who want to verify technical credibility.

### Actions (Week 2–3)

1. **Clean up GitHub profile**
   - Pin 2–3 relevant repositories
   - Write clear README descriptions emphasizing systems/workflow work
   - Remove or archive irrelevant repos

2. **Create one public workflow artifact**
   - Example: Open-source a workflow mapping template or decision routing pattern
   - Purpose: Technical proof that you can build, not just consult
   - Format: Well-documented repo with README, examples, and usage instructions

3. **Link from LinkedIn and website**
   - Add GitHub link to LinkedIn profile
   - Reference specific repos in proof-oriented LinkedIn posts

### GitHub Success Metric

- GitHub profile supports credibility when a buyer clicks through from LinkedIn or website
- No traffic targets — this is a validation tool, not a growth channel

## X / Twitter Strategy

### Phase 1 (Now): No Investment

Do not create original content for X. The audience overlap with the ICP (CIO/COO/Head of Ops) is too low compared to LinkedIn.

### Phase 2 (Month 2+): Repurposing Only

If LinkedIn content is performing well:

1. Cross-post the best LinkedIn post of the week to X
2. Use a slightly modified format (shorter, no line breaks, different hook)
3. Track engagement — only continue if ICP-relevant operators engage
4. Total time investment: 15 minutes/week maximum

### X Decision Rule

If after 4 weeks of repurposing, X produces zero relevant conversations or profile visits from ICP titles, stop entirely.

## Email / Newsletter Strategy

### Phase 1 (Now): No Investment

Do not start a newsletter until:
- Content inventory includes 20+ published LinkedIn posts
- A repeatable publishing rhythm exists (3/week for 4+ weeks)
- At least one lead magnet is live and generating downloads

### Phase 2 (Month 3+): Newsletter Launch

**Only when the above conditions are met:**

1. **Platform:** Resend + React Email (free tier, open source compatible) or Buttondown (simple, low-cost)
2. **Cadence:** Biweekly
3. **Content:** Curated from best LinkedIn posts + one original insight
4. **CTA:** Every issue ends with `Schedule a Discovery Call`
5. **List building:** Lead magnet downloads, LinkedIn bio link, website opt-in

### Newsletter Decision Rule

If after 8 issues the newsletter does not generate at least 1 discovery call or qualified DM, pause and redirect time to LinkedIn.

## Community Engagement Strategy

### Target Communities

Engage where the ICP congregates:

1. **LinkedIn Groups:** CIO/COO-focused groups, PMO communities, RevOps communities
2. **Slack Communities:** Operations-focused Slacks, data leadership communities
3. **Podcasts/Newsletters:** Guest opportunities (see Thought Leadership execution below)

### Community Engagement Protocol

- **Do:** Add value with specific insights, answer questions with experience-backed responses, share relevant content (yours and others')
- **Don't:** Pitch in community forums, drop links without context, spam groups

### Guest Appearance Targets

**Priority categories:**
- Operator and COO communities
- CIO and PMO audiences
- Data and analytics leadership newsletters
- AI in operations podcasts

**Outreach approach (Month 2+):**
1. Identify 10 podcasts/newsletters in target categories
2. Prepare a 3-topic pitch sheet (workflow bottlenecks, trustworthy AI, operator-led systems)
3. Send personalized outreach to hosts
4. Target: 1 guest appearance per month starting Month 3
